# Domain docs — consumer rules

## Layout

Single-context: one `CONTEXT.md` at the repo root, one `docs/adr/` directory.

## Rules for skills

1. Read `CONTEXT.md` before making architectural suggestions — it defines the project's domain language and constraints.
2. Check `docs/adr/` for past decisions before proposing changes that contradict them.
3. If `CONTEXT.md` does not exist yet, prompt the user to create one when domain context would improve a suggestion.
